創建事件議程文件

{{ page.description }}

通常，活動議程最初是在第三方工具中創建的，例如 Microsoft Excel 或 Google 表格。議程模塊要求上傳外部 .csv 議程文件以最初創建事件議程。議程文件上傳後，可以使用議程模塊對其進行編輯。

筆記：可以在此處下載示例議程模板文件。
筆記：無法在議程模塊中創建議程。您必須上傳 .csv 文件以初步填充活動議程。

列命名要求

創建初始議程文件時，您可以下載示例議程模板並對其進行編輯或使用您可能已創建的現有議程文件。Agenda 模塊查找特定列名稱的存在。這些列將映射到議程模塊中的相應字段。議程文件中可能存在的其他列將被議程模塊忽略。為了上傳議程文件，需要以下列：

  • name -會話名稱
  • start_date - 會話開始日期（MM/DD/YYYY 格式）
  • start_time -會話開始時間
  • timezone - start_time 的時區（使用來自IANA 時區 )
  • duration - 以分鐘為單位的會話持續時間
  • track - 識別軌道的價值（即技術、銷售、營銷）；曲目名稱全部表示會話跨越所有軌道（即午餐、休息）

以下列是可選的，但如果它們存在於您的議程文件中，它們也會被導入到議程模塊中：

  • description -會話描述
  • speaker1_first -演講者的名字
  • speaker1_last -演講者的姓氏
  • speaker1_email - 演講者電子郵件地址（用於唯一標識演講者）
  • speaker2_first -演講者2名
  • speaker2_last -演講者2的姓氏
  • speaker2_email - 演講者 2 電子郵件地址（用於唯一標識演講者）
  • speaker3_first - 演講者 3 的名字
  • speaker3_last - 演講者 3 的姓氏
  • speaker3_email - 演講者 3 電子郵件地址（用於唯一標識演講者）
  • session_type - 會議類型（會議、主題演講、會議、其他）
  • video _id - 視頻 ID（用於會議、主題演講和其他會議類型）
  • url - 會議 URL（適用於會議會話類型）
  • related _label - 將出現在議程模塊中的相關鏈接的標籤
  • related_link - 會話講義、幻燈片等的 URL 鏈接。
筆記：議程上傳最多支持三名發言人，可以使用議程模塊將其他發言人添加到會議中。

當您準備好將議程導入議程模塊時，將其導出為 .csv 文件。議程模塊只能導入 .csv 文件。有關如何將議程文件上傳到議程模塊的信息，請參閱上傳和自定義活動議程 .
