將視頻添加到虛擬活動體驗

{{ page.description }}

使用圖庫收藏編輯器，您可以選擇要在虛擬事件體驗上顯示的視頻。與其他門戶類型的體驗一樣，視頻也被組織成集合。

要將視頻添加到虛擬事件體驗中，請單擊影片在左側導航欄中。

視頻採集組織

集合的順序將控制它們在虛擬事件主頁上的顯示方式。

筆記：有關創建收藏夾和將視頻添加到收藏夾的信息，請參閱將視頻添加到門戶體驗.

虛擬事件模板有四個主要區域，按照示例站點中的指定顯示視頻：

  1. 精選集 -通常用於現場直播或模擬內容將在整個活動中顯示。理想情況下，應根據其從最早到最新的上線時間進行訂購，以獲得最佳體驗。
  2. 主要收藏 -可以包含任何內容，但通常用於特定的VOD突破或您希望引起更多關注的內容
  3. 次要收藏 -可以包含任何內容，但通常用於特定的VOD突破或您希望引起更多關注的內容
  4. 音軌 -通常用於VOD內容
筆記：Featured、Primary 和 Secondary 合集最多支持 24 個視頻。
筆記：要在主要/次要集合上方插入標題，請向配置文件中添加靜態內容對象。有關信息，請參閱使用靜態內容.

在Gallery中訂購收藏時，應按以下方式訂購收藏：

  • 集合1-精選集合
  • 收藏 2 - 主要收藏
  • 收藏 3 - 二次收藏
  • 集合4-曲目

配置事件跟踪

創建曲目時，請遵循以下準則，以確保它們正確顯示在體驗中：

  • 集合名稱將出現在虛擬事件模板中，因此請使用有意義的名稱
  • 曲目集合將是一個嵌套集合，該集合中不會出現任何視頻
  • 曲目收藏應該是第四個收藏
  • 在每個軌道的軌道集合下創建子集合
  • 創建收藏集時不要使用播放列表
  • 軌道部分的位置在配置文件中指定，可以改變通過編輯breakout_collection配置文件中的對象

配置曲目URL

創建集合後，需要更新配置文件，以便標頭中的按需會話鏈接將定向到正確的URL。用於軌道集合（集合4）的名稱將在URL中使用，以訪問軌道頁面。為確保正確設置URL，請確認以下內容。

  1. 查看配置文件並找到breakout_collection目的。
  2. 這collection參數包含軌道頁面的URL。例如：
    
"breakout_collection": {
    "collection": "/category/videos/tracks",
    "insert_target": ".bc-on-demand-section",
    "insert_mode": "after"
},
  3. 最後一條路collection URL應反映您的曲目集合（集合4）的名稱。例如，如果您的曲目收藏（收藏 4）被命名為“Sessions”，收藏參數將更新為：
    
"collection": "/category/videos/sessions",
  4. 要驗證配置，請單擊標題中的“按需會話"鏈接。曲目會話應出現。單擊一個曲目應打開一個曲目中的視頻。如果看到404錯誤，則說明存在配置問題。
