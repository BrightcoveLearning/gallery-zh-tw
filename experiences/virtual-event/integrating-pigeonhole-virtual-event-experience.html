將 Pigeonhole 與虛擬活動體驗相結合

{{ page.description }}

Pigeonhole 可用於輕鬆地將基於事件的交互性（實時問答、投票、調查等）添加到虛擬事件體驗中的一個或多個視頻。通常，為現場活動創建一個新的Pigeonhole，然後可以為要添加Pigeonhole功能的每個視頻創建會話。建立工作階段之後，iframe 嵌入程式碼會使用 Video Cloud 自訂欄位與每個視訊產生關聯。嵌入代碼將使Pigeonhole窗格顯示在所選視頻的視頻播放器旁邊。

創建一個新的Pigeonhole事件和會話

您應該為現場活動創建一個新的Pigeonhole，然後為每個添加了Pigeonhole功能的視頻添加會話。

  1. 登入您的信鴿洞帳戶。
  2. 點擊+添加鴿子洞然後點擊您的帳戶即可使用。
  3. 輸入一個活動名稱和密碼。
  4. 進入開始和結尾活動日期。這些日期控制鴿洞將活動多長時間。
  5. 選擇一個時區和定製品牌選項（如果適用）。
  6. 點擊繼續。
  7. 現在，事件已創建議程項目將在左側導航中被選擇。點擊新增工作階段然後點擊創建新的。
  8. 選擇一個會話類型的問與答（也可以使用其他會話類型）。
  9. 進入一個會話名稱。
  10. 進入開始和結尾日期和時間。
  11. 配置問答設置作為適當的。
  12. 點擊添加問答並確認會話已創建。
  13. 根據需要創建其他會話。

生成鴿孔嵌入代碼

為了讓Pigeonhole在視頻旁邊顯示，將使用“視頻雲"自定義字段將iframe嵌入代碼添加到視頻中。代碼的格式將是：

<iframe src="鴿巢網址"></iframe>

獲得價值鴿子URL， 按著這些次序。

  1. 在“鴿子洞"中，單擊舉辦活動在左側導航欄中。
  2. 點擊鏈接邀請您的參加者部分。
  3. 確認打開了一個帶有所有會話列表的新瀏覽器選項卡。
  4. 單擊相應會話的輸入鏈接，例如，輸入此問答。請注意，鏈接將根據會話類型而改變。
  5. 確認Pigeonhole在新的瀏覽器選項卡中打開。Pigeonhole URL顯示在瀏覽器地址欄中。

這iframe帶有Pigeonhole URL的代碼將被添加到適當的視頻中。

配置事件設置

還必須配置Pigeonhole事件的設置，以便可以使用SSO信息來創建用戶配置文件。

  1. 在左側導航中，單擊更多設置。
  2. 點擊參會者標籤。
  3. 檢查允許通過嵌入創建與會者個人資料選項。
  4. 向下滾動到註冊人資料頁面上的部分。
  5. 檢查啟用此事件的註冊者資料選項。

建立自訂欄位

為了使虛擬事件模板知道要為視頻顯示哪個Pigeonhole，請使用自定義字段。若要建立自訂欄位，請依照下列步驟執行。

  1. 登入視訊雲端工作室。
  2. 在導航標題中，單擊行政進而視頻場 .請注意，您必須是帳戶管理員才能存取 ADMIN 功能表。
  3. 創建一個新的自定義字段，名稱為聊天嵌入。
  4. 確認內部名稱為chat_embed。
  5. 設置類型成為文本。
  6. 保存自定義字段。

有關在Video Cloud中創建自定義字段的信息，請參閱創建自定義元數據字段。

在視頻中添加鴿子洞

若要讓 Pegeonhole 出現在虛擬事件範本中的視訊旁邊，請依照下列步驟執行。

  1. 開啟「 媒體」 模組。
  2. 按一下視訊以開啟視訊屬性。
  3. 找到自定義字段部分，然後單擊編輯。
  4. 將Pigeonhole嵌入代碼粘貼到聊天嵌入自定義字段。
  5. 點擊保存。

將用戶SSO信息傳遞給Pigeonhole

如果使用SSO保護了虛擬事件體驗，則可以將用戶SSO信息傳遞給Pigeonhole，以便可以在Pigeonhole內部跟踪用戶。以下各節詳細介紹了將SSO信息傳遞給Pigeonhole所需的設置和步驟。

映射SSO屬性

您需要確保映射了正確的SSO屬性，以便可以將它們傳遞給您的體驗，然後傳遞給Pigeonhole。積分Pigeonhole尋找以下值：

  • fname -用戶名
  • nameID -用戶的電子郵件地址

下面的示例顯示了從Okta SSO Admin頁面內部公開的這些屬性。

在訪問控製配置文件中公開屬性

確保用於確保體驗安全的訪問控製配置文件具有公開所有屬性選項已選中。如果 SSO 資訊無法使用，使用者會在 Pigeonhole 中顯示為匿名。

  1. 開啟「 圖庫」 模組。
  2. 點擊設定值在標題中。
  3. 在左側導航中，單擊訪問控製配置文件 .
  4. 按一下用來保護體驗的設定檔的編輯圖示。
  5. 確保暴露所有屬性設置被檢查。

有關配置單點登錄的信息，請參閱配置訪問控製配置文件。如果所有項目都設定正確，當檢視器登入體驗時，使用者資訊會顯示在 Pigeonhole 中。
