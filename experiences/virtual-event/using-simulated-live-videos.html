使用模擬直播視頻

{{ page.description }}

模擬直播是簡稱 模擬現場 並且是一種可以向最終用戶顯示 VOD 內容的方式，就好像它是實時的一樣。Simulive 內容對活動主辦者有吸引力，因為它允許在上線前預先錄製內容和廣泛的測試。它降低了由於編碼器問題或其他物流而發生的傳統現場事件的潛在故障日期可能產生的風險。Simulive 視頻將顯示一個倒數計時器與視頻，以指示它何時將被實時。下面的例子是特色集合中的模擬實況視頻。

對於主要和次要系列中的影片，倒數計時器會出現在影片的卡片上。

若要在虛擬事件體驗中使用模擬內容，必須執行以下作業：

  • 設定模擬播放器外掛程式
  • 將玩家與體驗建立關聯
  • 創建一個模擬的自定義字段
  • 設定模擬視訊屬性
筆記：作為倒數計時器一部分顯示的文本位於labels配置文件中的對象，並且可以輕鬆自定義或本地化。

配置模擬播放器插件

您應該為虛擬活動體驗中使用的玩家設定 Simulive 播放器外掛程式。聯繫您的客戶經理以獲取配置插件所需的信息。

  1. 點擊玩家們在導航標題中打開“播放器"模塊。
  2. Brightcove 建議您為您的虛擬活動體驗建立專屬的玩家。如果尚未創建專用播放器，請單擊新玩家，命名播放器，然後單擊保存。
  3. 按一下您要設定之玩家的連結。
  4. 點擊外掛程式在左側導航菜單中。
  5. 點擊添加插件>自定義插件。
  6. 為了插件名稱進入simulivePlugin。
  7. 為了JavaScript 網址和網址，使用您的客戶經理提供的值。
  8. 點擊保存保存更改。
筆記：對播放器所做的任何更改最多可能需要 5 分鐘才能顯示在已發布的播放器中。

設定自訂播放器

若要將虛擬事件體驗設定為使用自訂播放程式，請依照下列步驟執行。

  1. 打開畫廊模塊。
  2. 編輯體驗。
  3. 點擊視頻和播放>播放器和線索表在左側導航欄中。
  4. 選擇使用特定於站點的播放器選項，然後選擇配置有Simulive插件的播放器。
  5. 點擊保存。

創建一個模擬的自定義字段

自訂欄位是用來儲存視訊可用的日期和時間。要創建模擬自定義字段，請按照下列步驟操作。

  1. 在導航標題中，單擊行政進而視頻場 .請注意，您必須是帳戶管理員才能存取 ADMIN 功能表。
  2. 點擊添加自定義字段。
  3. 使用下列值建立新的自訂欄位。
    • 顯示名稱 -模擬開始
    • 內部名稱 -simulive_start
    • 類型 - 文本
    • 描述 -預期格式為YYYY-MM-DDTHH：MM：SS-HH：MM
  4. 點擊保存字段。
有關創建自定義字段的信息，請參見創建自定義元數據字段。

配置模擬視訊屬性

自定義字段模擬開始將保留視頻可用的日期和時間。在到達 Simulive 開始時間之前，影片將無法觀看。模擬視頻也必須加標籤模擬的。若要將視訊設定為模擬影片，請依照下列步驟執行。

  1. 開啟「 媒體」 模組。
  2. 按一下影片標題以開啟屬性。
  3. 找到視頻屬性部分，然後單擊編輯。
  4. 標記影片模擬的。
  5. 點擊保存。
  6. 找到自定義字段部分，然後單擊編輯。
  7. 輸入日期和時間值模擬開始場地。字段的正確格式是YYYY-MM-DDTHH:MM:SS-HH:MM在哪裡YYYY-MM-DD是年份，月份和日期，並帶有適當的前導零，HH:MM:SS是24小時，分鐘和秒，前導零是適當的，並且-HH:MM是時區偏移量（例如，對於UTC-4，是-04：00，對於UTC + 8.5，是+08：30，等等）。
    筆記：Simulive依靠給定用戶的本地計算機時間來確定事件應從何時開始，方法是將本地計算機時間與已配置的時間進行比較模擬開始時間。基於這個原因，某些情況下，某些使用者的事件可能會比預期更早啟動。這通常是因為這些使用者的' 電腦時間不正確。
  8. 點擊保存。

更新原則機碼值

為了讓倒數計時器顯示在模擬活影片上，組態檔案中的原則金鑰值必須使用已啟用搜尋的原則金鑰來更新。如果無法執行此動作，則影片不會顯示倒數計時。有關獲取啟用搜索的策略密鑰的步驟，請參閱代碼樣例：生成策略密鑰。創建新的策略密鑰後，請更新policyKey值在配置文件中，然後重新發布體驗。
