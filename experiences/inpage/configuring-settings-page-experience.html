頁面內容

    設定頁面內體驗的設定

    在本主題中，您將學習如何自訂頁面內體驗的設定。

    這些設定可控制頁內體驗的各個層面。您可以使用下列設定：

    • 玩家
    • 視訊
    • 社交
    • 搜索引擎優化
    • 追蹤
    • 高級
    注意：對體驗進行任何變更之後，請確定您重新發佈體驗，以便變更會出現在顯示體驗的網站上。

    設定播放器設定

    播放程式設定控制所使用的播放器，以及該體驗的播放程式行為。要配置播放器設置，請編輯體驗，然後單擊設置>播放器在左側導航欄中。

    選擇播放器

    要選擇用於頁內體驗的播放器，請從可用球員下拉列表。清單會顯示您的視訊雲端帳戶中的所有玩家。

    選擇播放器後，播放器預覽將顯示所選播放器的預覽。

    注意：使用頁面內體驗時，播放器樣式只會在影片播放時反映出來。玩家樣式將不會在「播放前」或「播放之後」狀態中看到。

    設定播放器行為

    玩家設定可以設定為自訂播放器的行為。您可以使用下列設定：

    • 視頻播放完成後確定視頻播放完畢後會發生什麼：
      • 顯示“播放後”狀態以進行體驗 -選擇後，體驗將進入比賽后影片播放完畢時的狀態。
      • 保留在當前視頻上 -選擇後，體驗將保留在玩遊戲影片播放完畢時的狀態。選取此選項可顯示為播放程式設定的自訂終端螢幕 (在 Player 模組中設定)。期間顯示的組件玩遊戲狀態也將保持可見。此選項的副作用是有效刪除比賽后從經驗中陳述。
    • 在 iOS 裝置上啟用內嵌播放功能-如果核取此選項，則在使用 Brightcove Player 5.10.5 或更新版本時，可在 iOS 裝置上進行內嵌播放。
    • 自動前進到下一個視頻 -如果選中此選項，則噹噹前視頻播放完畢時，將播放體驗中的下一個視頻。此設置還確定是否在計時器中顯示倒數計時器比賽后州。如果啟用，將會有一個計時器。當計時器達到零時，體驗將過渡到玩遊戲下一個視頻的狀態。

    使用 IP 限制帳戶

    IP限制帳戶北美以外的地區需要修改Playback API base_url在播放器中。頁內體驗將復制此內容base_url在創建體驗以及為體驗選擇播放器時從播放器獲取。如果base_url如果更改了現有體驗的播放器，Gallery將不會自動更新此值，因為我們不知道它已更改。作為因應措施，發行者應該遵循下列步驟作為因應措施：

    1. 前往「播放器設定」頁面以取得體驗。
    2. 選取不同於目前選取的播放器。
    3. 重新選擇初始玩家。
    4. 儲存並發佈體驗。

    這將更新base_url以獲得與初始玩家相匹配的體驗。

    完成後，按一下「儲存」以儲存變更。

    使用受限網域限制的玩家

     受域限制的播放器需要修改策略密鑰 在播放器中。頁內體驗將復制此內容policy_key 在創建體驗以及為體驗選擇播放器時從播放器獲取。如果 policy_key 如果更改了現有體驗的播放器，Gallery將不會自動更新此值，因為我們不知道它已更改。因應措施，如果發行者更新核可網域的清單，發行者應該遵循下列步驟作為因應措施：

    1. 前往「播放器設定」頁面以取得體驗。
    2. 選取不同於目前選取的播放器。
    3. 重新選擇初始玩家。
    4. 儲存並發佈體驗。

    這將更新policy_key 以獲得與初始玩家相匹配的體驗。

    完成後，按一下「儲存」以儲存變更。

    設定視訊設定

    視訊設定可控制某些視訊顯示選項，並控制視訊是否可以下載。這些設置僅在視頻位於玩遊戲州。要配置視頻設置，請編輯體驗，然後單擊設置>視頻在左側導航欄中。

    相關連結

    勾選此選項後，視訊的相關連結將會顯示視訊資訊。相關鏈接被配置為媒體模塊中的視頻屬性。

    描述

    如果勾選此選項，則會顯示視訊的完整說明，而不會顯示視訊資訊的簡短說明。簡短說明和詳細說明被配置為媒體模塊中的視頻屬性。

    已觀賞標示

    如果選中，則為觀察指示器（）將出現在用戶會話期間觀看的視頻中。

    注意：觀看指示器將使用疊加橫幅樣式菜單中的屬性。

    下載

    您可以設定頁面內體驗，讓觀眾可以下載影片。下載連結可讓檢視者下載最高品質的轉譯。選擇下載選項：

    • 不允許下載視訊
    • 提供每段視訊的下載連結
    • 根據自訂欄位值為每個視訊提供下載連結-定義自訂欄位，然後針對任何可下載的視訊，將該欄位設定為 true。有關為每個視頻配置下載鏈接的完整步驟，請參見配置視頻下載以實現頁內體驗。

    完成後，按一下「儲存」以儲存變更。

    設定社交設定

    社交設定可控制影片播放時可用的社交分享選項。體驗進入遊戲後，社交共享圖標會顯示在播放器上方。玩遊戲州。將影片分享至社交媒體網站時，會出現連結或影像，讓觀眾回到體驗。

    圖庫使用社交媒體中繼資料代理服務。每當從網頁內體驗分享影片或活動時，都會使用此服務，並且是共享網址的一部分。它會將檢視者重新導向至頁面內體驗嵌入的頁面，但是，檢索程式會看到有關視訊/事件的中繼資料。通過這種方式，社交媒體爬蟲將獲得有關被共享的視頻/事件的信息，並可以在社交媒體網站上向最終用戶提供更多相關的信息，而最終用戶仍然能夠訪問原始頁面。

    要配置社交設置，請編輯體驗，然後單擊設置>社會在左側導航欄中。

    注意：在社交媒體中編輯體驗時，也可以配置社交設置佈局通過點擊配置共享鏈接。

    顯示

    「顯示」設定決定社交圖示的顯示方式。當。。。的時候分享選項 （）被選中，共享圖標將從播放器右側滑入。

    當社交圖標選項（）被選中，社交圖標將顯示在播放器上方。

    分享選項

    選取要顯示的社交分享選項。某些平台提供輸入自訂訊息的能力。按一下社交平台名稱以輸入自訂訊息。按一下連結，即可使用訊息中的影片 URL、名稱或簡短描述值。

    再次按一下社交平台名稱以隱藏自訂訊息設定。

    分享連結用的社交中繼資料

    輸入頁面網址嵌入體驗的頁面中的，然後單擊生成。這可讓社交網站在觀眾分享您的體驗時，顯示第一部影片的縮圖影像、標題和說明。

    存取 SEO 設定

    SEO 設定可讓您產生網站地圖和搜尋頁面內體驗的摘要。要訪問SEO設置，請編輯體驗，然後單擊設置> SEO在左側導航欄中。

    若要產生網站地圖和/或搜尋摘要，請依照下列步驟執行：

    1. 輸入頁面網址。這是將發布頁面內體驗的網站 URL。
    2. 檢查視頻標籤是否應包含在內。
    3. 按一下「產生」。

    網站地圖程式碼應該複製到您現有的網站地圖，以索引頁面內體驗資訊。

    若要設定入口網站體驗的搜尋引擎最佳化設定，請編輯體驗，然後按一下左側導覽中的 [ 網站設定 > 搜尋引擎最佳化 ]。您可以使用下列設定：

    • 網站詳細資料 -設定網站映像、關鍵字和中繼標記
    • 網站地圖 -訪問網站地圖和搜索摘要 URL

    設定網站詳細資料

    您可以設定「網站詳細資料」設定，以控制搜尋引擎建立入口網站索引的方式。若要存取「搜尋引擎最佳化」設定，請編輯體驗，然後按一下左側導覽中的「網站設定 > 搜尋引擎最佳化」。

    您可以設定下列設定：

    • 在搜尋引擎結果中使用網站說明，當網站分享時-勾選此選項時，會使用中 < 繼 > 描述標記中的網站說明，包括 Facebook 和 Twitter 中繼標記。如果未勾選，則會使用精選影片的說明。
    • 網站圖片 -這是當一個門戶網站的主頁被分享在 Facebook 或 Twitter 上顯示的圖像。
    • 關鍵字 -將被添加到所有網站頁面的元關鍵字。添加「鳥類，動物，魚，動物世界」將導致以下內容被添加到頁面 HTML：
      <meta name="keywords" content="birds,animals,fish,animal world">
    • 中繼標籤 -用於新增網站的其他中繼標記。這些標籤將被添加到所有網站頁面。範例：
      <meta name="myCustomTag" content="myCustomValue">
    • 在頁面中繼資料中包含自訂視訊欄位 -勾選此選項時，會將自訂欄位新增為中繼標籤。例如，一個值為 2018 年的自定義字段將添加：
      <meta name="year" content="2018">
    • 在頁面元數據中包含規範 URL -選中時，將在源代碼中添加一個rel=canonical元標記，這有助於提高 SEO。這是一個功能，告訴谷歌將具有相同rel=canonical鏈接的所有網頁視為一個條目。例如，我的網站/視頻/Myypag.html 與我的網站/視頻/我的頁面相同。
    • 從網址移除影片標題 -勾選此選項後，視訊名稱將不會包含在影片詳細資料頁面的 URL 中，只有影片 ID 會出現在 URL 中。

    查看網站地圖和搜索摘要

    網站地圖設定會顯示網站地圖和搜尋摘要的 URL。網站地圖可以下載並提交給搜索引擎。大部分的搜尋引擎軟體都可以使用搜尋摘要，作為將「圖庫」內容新增至搜尋結果的方式。若要存取「搜尋引擎最佳化」設定，請編輯體驗，然後按一下左側導覽中的「網站設定 > 搜尋引擎最佳化」。

    若要在網站地圖中包含視訊標籤，請勾選「在網站地圖中包含視訊標籤」選項。請注意，只包含前 32 個標籤。在網站地圖中加入標籤會將標記公開至搜尋引擎網頁檢索器，讓訪客可以根據影片標記來尋找影片。

    注意：在網站發佈之前，這些 URL 將無法使用。

    網站地圖會快取 24 小時。24 小時後，它們會從快取中移除，並在再次存取時重新產生。如果影片已新增至入口網站體驗，在重新產生網站地圖 (24 小時後) 或網站重新發佈之前，這些影片將不會包含在網站地圖中。

    如需建立提交網站地圖的詳細資訊，請參閱 Google 文件。

    設定追蹤

    追蹤設定可用於將追蹤資料傳送至 Google Analytics (分析)。要配置跟踪設置，請編輯體驗，然後單擊設置>跟踪在左側導航欄中。

    透過在追蹤頁面中提供 Google Analytics 追蹤 ID，在體驗中發生的事件將會提交至 Google Analytics (分析) 帳戶。每個事件都有一個類別，形式為Brightcove體驗{name}-{state}哪裡{名稱}是體驗的名稱，並且{州}是組件的當前播放狀態（之前，期間，之後）。每個事件也可能有一個相關的標籤，這通常是發生事件的視頻的 ID。目前會追蹤下列事件 (未來會視需要新增更多事件)：

    動作 描述
    點選精選影片 精選視頻被點擊
    點選視訊 按一下影片
    已點選觀看的影片 已點選觀看的影片
    點選下一個視訊 下一個視頻被點擊
    按一下重新執行 點擊「重播」按鈕
    自動提前停止 下一個視頻倒計時停止/取消
    自動進階倒數經過，播放下一部影片 下一個影片倒數計時器過去，所以播放下一部影片
